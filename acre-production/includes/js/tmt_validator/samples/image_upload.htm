Image upload validation

Client side validation for image upload can be somewhat tricky. Due to some exaggerated security concerns this will not work with Firefox 1.0+ and Mozilla 1.7+ unless the script is called directly from the local system. This function will fail silently on these Gecko based browsers.

It has been fully tested in IE 5.5+ and works in Safari and IE 5.2 on the Mac.

In order to enable image validation, we add the tmt:image attribute to an input/file field. A simple example looks like this:

<input type="file" name="image_upload" tmt:message="Max width 360px. Max size 50kb." tmt:image="true" tmt:imagemaxwidth="360" tmt:imagemaxsize="50" />

Image upload validation works with .jpg and .gif files. It accepts up to 5 different attributes that can check both file size and dimension:

  • tmt:imagemaxsize: maximum file size (kb)
  • tmt:imagemaxwidth: maximum width (px)
  • tmt:imageminwidth: minimum width (px)
  • tmt:imagemaxheight: maximum height (px)
  • tmt:imageminheight: minimum height (px)

In order to avoid having images with spaces or special characters inside their names, we can use one of the built-in patterns (see sample) or define a custom one (see sample).

The example below shows all the validation rules and a pattern mix for image upload:

<input type="file" name="jpg_upload" tmt:pattern="filepath_jpg" tmt:message="Please select a .jpg file. Max width 360px. Min width 240px. Max height 240px. Min height 180px. No white space or special chars inside its name." tmt:image="true" tmt:imagemaxwidth="360" tmt:imageminwidth="240" tmt:imagemaxheight="240" tmt:imageminheight="180" />
