When things go wrong...

Since it makes heavy use of DOM, XHTML and JavaScript, everything inside the TMT Validator is case sensitive. The whole library has been designed with power users in mind, so it doesn't tolerate sloppy coding (most of the time it will fail silently instead of throwing JavaScript errors)

The submit button's value isn't passed to the server-side script

By default submit buttons are disabled after the form is submitted. This prevents multiple submits but also prevents the value of the submit buttons from being passed as part of the HTTP request. Set the tmt:blocksubmit attribute of the <form> tag to false if you want prevent this behavior

Conflicting JavaScript code on page load

The TMT Validator needs to be initialized after the document is loaded. The library internally achieves this by using the JavaScript code below:

addLoadEvent(tmt_validatorInit);

More details about this handy piece of code are available here: http://www.sitepoint.com/blog-post-view.php?id=171578 (thanks to Danilo Celic)

Problems can easily arise if, inside your XHTML code, you are using the onload attribute of the <body> tag like:

<body onload="doThis()">

This will prevent the validator's from initializing, and the whole validation process will never happen. There are different ways to solve this.

An easy, but less elegant solution is to add a call to tmt_validatorInit inside your <body> tag:

Solution n°1
<body onload="doThis();tmt_validatorInit()">

If you prefer to minimize the amount of JavaScript calls in your XHTML tags (a worthwhile goal), you can use something like:

Solution n°2
<head>
<script src="script_tmt_validator.js" type="text/javascript"></script>
<script language="JavaScript">
function doThis(){
// Your JS code here
}
addLoadEvent(doThis);
</script>
</head>
<body>

As an alternative, you can assign an anonymous function to the window's onload event:

Solution n°3
<head>
<script src="script_tmt_validator.js" type="text/javascript"></script>
<script language="JavaScript">
function doThis(){
// Your JS code here
}
window.onload = function(){
doThis();
tmt_validatorInit();
// More JS code here if you need
}
</script>
</head>
<body>

The TMT Validator tries to be as unobtrusive as possible and to play nicely with other scripts, so something like this will work just fine:

<script language="JavaScript">
function doThis(){
// Your JS code here
}
window.onload = doThis;
</script>
<script src="../script_tmt_validator.js" type="text/javascript"></script>

The code below will prevent the validator from working because "doThis" takes full control over the onload event. Note that the TMT Validator scripts have been included before our doThis() function instead of after it.

<script src="../script_tmt_validator.js" type="text/javascript"></script>
<script language="JavaScript">
function doThis(){
// Your JS code here
}
window.onload = doThis;
</script>

Any of the 3 solutions above will work just fine to fix this. Just pick the one that fits your needs and coding habits best.

About the onsubmit event of the <form> tag

The TMT Validator takes full control over the onsubmit event of the form tag. So, if you have something like this:

<form action="index.htm" method="post" tmt:validate="true" onsubmit="doThis()">

The call to "doThis()" will be overwritten by the Validator's parser and never executed.

To fix this you have to open the script_tmt_validator.js file and hack it a little bit (don't try this unless you have good knowledge of JavaScript). First of all, locate the tmt_validatorInit function (it's at the very top of the file) and comment three lines of code:

// Set the form node's onsubmit event
//formNodes[i].onsubmit = function(){
// return tmt_validateForm(this);
//}

Then modify the XHTML code:

<form action="index.htm" method="post" tmt:validate="true" onsubmit="doThis();return tmt_validateForm(this)">

Be advised this is a pure hack; please consider all its implications before start using it. It will make your code a little harder to maintain and you will have to apply the hack again before using any updated version of script_tmt_validator.js.

Filters conflicting with onblur and onkeyup

If you use tmt:filters on text field, you can’t call custom JavaScript code using their onblur and onkeyup events; since, in order to make the filters work, the Validator takes full control of the two events.

Hopefully this problem will be addressed during a future release.

Namespaces and Gecko based browsers

If you are serving your pages using an XML mime-type, and you are getting this weird error from the most recent versions of Gecko based browsers:

XML Parsing Error: prefix not bound to a namespace

...all you need to do is to add the proper namespace declaration inside the <html> tag:

<html xmlns:tmt="http://www.massimocorner.com/validator" xmlns="http://www.w3.org/1999/xhtml">
